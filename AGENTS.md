# Agent Instructions

## Commit Messages

- Always use Conventional Commits for every `git commit`.
- Format: `<type>(<scope>): <description>`
- Allowed `type` values: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`.
- Keep subject line imperative and concise.

Examples:
- `feat(auth): add oauth callback validation`
- `fix(web): resolve admin import path alias`
- `chore(repo): rename package directory to convex-zen`
